# Claude's Memory

This directory is my persistent memory for the Wedding Website project. The execution
environment is wiped between sessions, so **this is how knowledge survives** — it lives
in the repo and is committed with my work.

See `CLAUDE.md` section 5 for the full memory discipline. Quick map:

- **`wedding-details.md`** — canonical wedding content (source of truth for the site)
- **`decisions.md`** — design & technical decisions, with rationale
- **`changelog.md`** — dated log of what changed each session
- **`tasks.md`** — roadmap, backlog, open questions
- **`learnings.md`** — lessons, gotchas, client preferences

Rule of thumb: if I would be annoyed to have to figure it out again next session,
it belongs here.
